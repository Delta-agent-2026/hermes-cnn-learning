Output Tokens

The original Transformer (Vaswani et al., 2017) uses a stack of 6 encoder layers and 6 decoder layers. Each layer contains residual connections and layer normalization. The key innovation is self-attention, which lets every token attend to every other token in the sequence.

Core Mechanism

Multi-Head Self-Attention

How It Works

Each token produces three vectors: Query (what am I looking for), Key (what do I contain), and Value (what I actually provide). The attention score between any two tokens is computed as the dot product of their Query and Key vectors, scaled by the square root of the dimension.

Attention(Q, K, V) = softmax(Q · Kᵀ / √dₖ) · V

Multi-Head Variant

Instead of computing attention once with a single head, we learn h different projection matrices (head-specific Q, K, V), compute attention in parallel on each head, concatenate the results, and project again.

MultiHead(X) = Concat(head₁, ..., headₙ) · WO
where headi = Attention(X · WQi, X · WKi, X · WVi)

Typical config: d_model = 512, h = 8, dk = dv = 64
Building Blocks

Encoder Layer

Stack Structure

Encoder(x) = Sublayer(x + LN(Sublayer(x)))

Sublayer₁ = MultiHeadSelfAttention(x)
Sublayer₂ = PositionWiseFFN(x)

Each sublayer has two key properties: (1) Residual connections around it (x + sublayer) and (2) Layer Normalization after each addition. This deep residual path enables gradient flow through all 6 encoder layers.

Position-wise FFN

FFN(x) = max(0, x · W₁) · W₂ + b₂

Two linear layers with ReLU activation. Widened internally (d_ff = 2048 in original).

Positional Encoding

PE(pos, 2i) = sin(pos / 10000^(2i/d))
PE(pos, 2i+1) = cos(pos / 10000^(2i/d))

Sinusoidal positions encode relative position without parameters. Different frequencies per dimension.

Sequence Generation

Decoder Layer

Three Sublayers

Decoder(x) = Sublayer₁(x + LN(Sublayer₁(x)))
          Sublayer₂(x + LN(Sublayer₂(x)))
          Sublayer₃(x + LN(Sublayer₃(x)))

Sublayer₁: Masked MHA

Self-attention where each position can only attend to earlier positions (causal mask). Prevents seeing future tokens during training.

Sublayer₂: Cross Attention

Queries from decoder, Keys/Values from encoder output. Bridges encoder and decoder — "what from the input should I attend to?"

Sublayer₃: FFN

Same position-wise FFN as the encoder. Processes the attended information into a representation for the next layer.

Causal Mask Example

With mask = -∞ for future positions, softmax of those becomes 0, effectively blocking them:

Position 1: [1, 0, 0, 0, 0] (can attend to itself only)
Position 2: [1, 1, 0, 0, 0] (self + position 1)
Position 3: [1, 1, 1, 0, 0] (self + first two)

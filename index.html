Core Concept

What Is a Convolution?

A convolution is a mathematical operation where a kernel (also called a filter) slides across an input matrix, computing element-wise products and summing them at each position. In CNNs, this operation detects local patterns like edges, textures, and shapes.

Key Properties

Stride

How many pixels the kernel moves at each step. Stride=1 moves one pixel. Stride=2 jumps two. Larger stride = smaller output.

Padding

Extra border of zeros around the input. "Same" padding keeps the output the same size as input. "Valid" means no padding.

Channels

RGB images have 3 channels. Each channel gets its own kernel. Output channels come from different kernels applied to all input channels.

Learnable Parameters

Weights & Biases

Weights determine how strongly each input feature influences the output. Biases shift the activation function, allowing the model to fit data that doesn't pass through the origin. During training, backpropagation adjusts both.

Pre-activation (logit): z = w₁x₁ + w₂x₂ + ... + wₙxₙ + b
ReLU activation: a = max(0, z)

How Weights Learn

Forward pass: Compute z = wx + b, then apply activation (e.g., ReLU: max(0, z)).
Loss: Compare the prediction to the true label using cross-entropy or MSE.
Backward pass: Compute gradients ∂L/∂w and ∂L/∂b using the chain rule.
Update: Adjust parameters: w = w − η·∂L/∂w, where η is the learning rate.

z = 0.5
output = ReLU(0.5) = 0.5
Pre-Softmax Scores

What Are Logits?

Logits are the raw, unnormalized scores output by the final layer of a neural network — before softmax is applied. They can be any real number (positive or negative) and don't sum to 1.

Logit = raw score from the network (no activation)
Probability = softmax(logit) → values in [0, 1] that sum to 1
Prediction = argmax(probability) → class with highest probability

Logits vs Probabilities

Logits are arbitrary real numbers. Softmax converts them into proper probabilities. See the difference below:

Class Logit Raw exp(z) Softmax P

Toy Example: Image Classification

Suppose a CNN classifies images as cat, dog, or bird. The last layer outputs 3 logits:

zcat = 2.0, zdog = 1.0, zbird = 0.5

Before softmax, the network "prefers" cat most (highest logit). But logits aren't probabilities — they could be negative, and they don't sum to 1. Softmax fixes that.
Normalization Function

The Softmax Function

Softmax converts raw logits into a probability distribution. It exponentiates each logit (so all outputs are positive) and normalizes so they sum to 1. This makes the output interpretable as class probabilities.

softmax(zᵢ) = ezᵢ / Σ ezⱼ   (for all classes j)

Why Softmax Matters

Strength Effects

Softmax amplifies differences between logits. A small gap like 2.0 vs 1.0 becomes 73% vs 27%. A larger gap like 5.0 vs 0.0 becomes 99.3% vs 0.07%. This is why high-confidence predictions can be overconfident.

Temperature Scaling

Dividing logits by a temperature T before softmax softens or sharpens the distribution. High T → uniform probabilities (more uncertain). Low T → sharper peaks (more confident). T = 1 is standard.

Example 3: Full Forward Pass (Logits → Softmax → Prediction)

Last layer logits:

z_dog = 1.2
z_bird = 0.1
Apply softmax:
e^2.5 = 12.182
e^1.2 = 3.320
e^0.1 = 1.105
sum = 16.607

P(cat) = 12.182 / 16.607 = 0.7335 (73.4%)
